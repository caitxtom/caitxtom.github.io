XQuery, shorthand for XML Query Language, is to XML data what SQL is to databases. Its main function is to query XML data, meaning it finds and extracts data (elements and attributes) from XML documents.

Take the files hw8.xml and hw8.xq for example. The XQuery file hw8.xq is the query to find and return all book titles with a price less than $30 from hw8.xml. Here is a breakdown of each line of code:

The for clause selects all book elements under the bookstore element into a variable called $b. The where clause acts as a filter that selects price elements from book elements and a price element with a value less than 30. Again, the variable $b holds all book elements' information. Finally, the return clause specifies what elements or data should be extracted from the document. Here it returns the title elements. The result would be

                    <title lang="en">Harry Potter</title>
                    <title lang="en">The Snowy Days</title>

If there was no specification for the title element, the result would include the book element and all elements under it:

                    <book category="children">
                        <title lang="en">Harry Potter</title>
                        <author>J K. Rowling</author>
                        <price>25.00</price>
                    </book>
                    <book category="children">
                        <title lang="en">The Snowy Days</title>
                        <author>Ezra Keats</author>
                        <price>5.89</price>
                    </book>

An additional part of XQuery is XQIB, or XQuery In the Browser, which allows you to program client-side web applications with XQuery. It is meant to be used instead of or alongside JavaScript. An example from the official website http://www.xqib.org/ shows how it can be used with the following source code:

                    <!DOCTYPE html>
                    <html xmlns="http://www.w3.org/1999/xhtml">
                        <head>
                            <title>XQIB: Sample page</title>
                            <meta charset="UTF-8"/>
                            <script type="text/javascript" src="mxqueryjs/mxqueryjs.nocache.js"></script>
                            <script type="application/xquery">
                                b:alert("Hello, world!")
                            </script>
                        </head>
                        <body>
                            <h1>Hello world page.</h1>
                        </body>
                    </html>

In order to for the XQuery to be executed and run in the browser, the script with the .js source launches the MXQuery engine. It is important to note that those files must be downloaded to the same server. In short, XQIB allows you to execute XQuery code, just as with JavaScript.
